# 数据融合与可视化方法

## 一、分层建模

EvoAtlas 将数据拆成六层：

1. **名称层**：接受名、异名、中文名、作者与命名年份。
2. **分类层**：域、界、门、纲、目、科、属、种等等级关系。
3. **系统发育层**：共同祖先、分叉、多分叉、备选拓扑和支持度。
4. **时间层**：分化时间的点估计、区间、方法和参考研究。
5. **化石层**：首次/末次出现、地层、地点、标本和鉴定历史。
6. **知识层**：性状、生态、分布、基因组、图片和保护信息。

分类树与系统发育树应分别存储。传统分类等级是人为命名体系，系统发育拓扑是对演化关系的科学推断，两者可能不完全一致。

## 二、推荐 ETL 流程

```text
下载版本化快照
  ↓
名称标准化与异名合并
  ↓
外部 ID 对齐和冲突队列
  ↓
导入综合系统发育拓扑
  ↓
叠加分化时间及区间
  ↓
叠加化石出现记录
  ↓
补充性状、媒体和中文名称
  ↓
生成按类群裁剪的前端快照
  ↓
质量检查与版本发布
```

## 三、冲突处理

- 同一名称对应多个概念：保留 `taxon_concept` 与来源版本，不直接覆盖。
- 来源之间拓扑冲突：保存备选边集合和支持文献，界面显示“存在争议”。
- 分化时间差异：显示分布或区间，允许切换不同研究综合方案。
- 中文名冲突：保存地区、来源和适用范围，不生成无来源的机器翻译作为正式名称。
- 化石鉴定变更：保留原始鉴定和当前鉴定。

## 四、前端可视化

### 时间树

横轴表示距今年代，纵轴只用于展开分支，本身没有“高低等级”含义。深时跨度很大，默认使用对数尺度，同时提供线性尺度避免用户误解。

### 多尺度聚合

- 世界尺度：域、超类群、界和主要门。
- 类群尺度：纲、目、科。
- 局部尺度：属和种。
- 节点密集时使用聚合节点，显示后代数量与抽样规则。

### 网络补充

以下关系不适合仅用树表示：

- 原核生物水平基因转移
- 植物杂交与多倍化
- 古人类基因渗入
- 病毒重配与重组
- 内共生起源

正式版应提供“树视图”和“网状演化视图”。

## 五、数据库建议

核心关系可用 PostgreSQL 存储，使用邻接表和物化路径支持分类路径；大规模图查询可引入 Apache AGE 或 Neo4j。推荐表：

- `taxon_concept`
- `taxon_name`
- `taxonomy_edge`
- `phylogeny_node`
- `phylogeny_edge`
- `divergence_estimate`
- `fossil_occurrence`
- `trait_observation`
- `media_asset`
- `external_identifier`
- `source_release`
- `curation_issue`

## 六、质量指标

- 名称匹配率与无法匹配队列
- 节点外部 ID 覆盖率
- 有时间区间节点比例
- 有文献支持边比例
- 未解析多分叉数量
- 争议拓扑数量
- 图片授权完整率
- 中文名带来源比例
- 数据版本更新延迟
