# EvoAtlas · 生物分类进化图谱

EvoAtlas 是一个无需构建工具即可运行的交互式“生命之树”网站原型。它把分类层级、系统发育拓扑、分化时间、代表性化石和数据置信度放到同一张图中，并提供任意两个物种的最近共同祖先比较。

## 已实现

- 按地质时间排列的系统发育树，支持拖拽、缩放和适应画布
- 对数时间尺度与线性时间尺度切换
- 全生命、植物、动物、脊椎动物、哺乳动物、灵长类、人类谱系等聚焦视图
- 中文名、学名和分类等级搜索
- 现生类群、灭绝类群、祖先/干群节点的区别显示
- 奥陶纪末、晚泥盆纪、二叠纪末、三叠纪末、白垩纪末事件背景
- 节点详情：分类路径、年代、关键特征、来源和置信度
- 两个物种的最近共同祖先与分支路径比较
- 深色/浅色主题和移动端适配
- 106 个教学型示范节点，含 16 个代表物种和 5 个化石类群
- 无第三方前端依赖，可直接静态部署

## 运行

直接打开 `index.html` 即可。为避免部分浏览器的本地文件限制，推荐启动一个静态服务器：

```bash
python3 -m http.server 8000
```

访问 `http://localhost:8000`。

## 数据定位

当前数据位于 `data/data.js`，是为验证交互设计而人工整理的代表性子集，不是完整物种名录。深时年代使用近似值，正式版应通过版本化数据管线自动获取并校验：

| 数据层 | 推荐来源 | 用途 |
|---|---|---|
| 接受名、异名、等级 | Catalogue of Life、GBIF | 名称归一化与分类主干 |
| 系统发育拓扑 | Open Tree of Life | 综合树与多分叉关系 |
| 分化时间 | TimeTree | 分子钟研究综合值和范围 |
| 基因组关联 | NCBI Taxonomy | Taxonomy ID、基因组与序列 |
| 化石记录 | Paleobiology Database | 地层范围、出现地点与化石类群 |
| 性状与生态 | Encyclopedia of Life | 形态、食性、栖息地和相互作用 |
| 图片与许可 | Wikimedia Commons | 图片、作者、许可证与原始文件页 |

详见：

- `docs/data-sources.md`
- `docs/methodology.md`
- `docs/roadmap.md`

## 数据模型

每个节点至少包含：

```js
{
  id: "sapiens",
  parent: "homo",
  zh: "智人",
  latin: "Homo sapiens",
  rank: "种",
  ageMa: 0,
  clade: "human",
  status: "extant",
  confidence: "high",
  summary: "...",
  traits: ["..."],
  sources: ["gbif", "ncbi", "col"]
}
```

灭绝类群还可包含 `firstMa` 和 `lastMa`。正式数据库还应增加稳定外部 ID、数据版本、引用文献、备选拓扑、时间区间、媒体授权、地理分布和保护状态。

## 数据校验

```bash
python3 scripts/validate_data.py
```

校验内容包括：重复 ID、缺失父节点、循环引用、状态字段、时间范围和父子年代关系。

## 部署

项目是纯静态站点，可直接拖入 Netlify，也可连接 GitHub 仓库。仓库已包含 `netlify.toml`：

```toml
[build]
  publish = "."
```

## 科学表达原则

1. 现生物种彼此通常是亲缘分支，不是“高级物种由低级物种变成”。
2. 分叉点表示共同祖先假设，不等于发现了一个确定的祖先个体。
3. 系统发育树是随证据更新的模型，争议节点必须显示置信度和版本。
4. 未解析关系使用多分叉，不为视觉整齐而随意二叉化。
5. 水平基因转移、杂交和基因渗入需要网络视图补充单棵树。
6. 化石类群默认作为干群、旁支或近缘代表，不自动标记为直系祖先。

## 后续工程化建议

正式版可改为 React + TypeScript + Canvas/WebGL 渲染，后端使用 PostgreSQL + Apache AGE 或 Neo4j 存储图结构；使用离线 ETL 生成版本化快照，前端只读取裁剪后的子树和聚合层级，避免一次加载数百万节点。
